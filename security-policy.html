1. Purpose

This policy establishes the information security framework for DragonBall ERP, a cross-border e-commerce management platform integrated with TikTok Shop. It ensures the confidentiality, integrity, and availability of all data processed by the system.

2. Scope

This policy applies to all systems, applications, and data managed by DragonBall ERP, including but not limited to product data, order information, financial records, and user credentials obtained through TikTok Shop API integration.

3. Data Classification

Classification Examples Protection Level
Public Product titles, product images Standard storage
Internal Order data, financial records, store analytics SSL-encrypted database, access-controlled
Confidential API keys, OAuth tokens, user passwords AES-256 encryption / bcrypt hashing

4. Access Control

  • Principle of Least Privilege: Users are granted the minimum access necessary.
  • Role-Based Access Control (RBAC): Three tiers — Member, Admin, Owner.
  • Multi-Tenant Isolation: Each organization's data is strictly isolated via tenant ID filtering.
  • Authentication: Passwords hashed with bcrypt (salt rounds: 12). JWT-based session tokens with expiration.

5. Data Encryption

  • In Transit: All communications use HTTPS with TLS 1.2 or higher.
  • At Rest: Database connections use SSL encryption. Sensitive fields use AES-256 encryption.
  • Credentials: API keys and OAuth tokens stored encrypted. User passwords stored as bcrypt hashes.

6. Vulnerability Management

  • GitHub Dependabot monitors dependency vulnerabilities automatically.
  • Regular npm audit scans for known security issues.
  • Code reviews are required before production deployments.
  • Critical vulnerability patches are deployed within 24 hours.

7. Incident Response

Phase Timeline Action
Detection Immediate Automated monitoring alerts technical lead
Assessment Within 1 hour Evaluate scope and severity of the incident
Notification Within 24 hours Notify affected users and TikTok platform
Remediation Within 72 hours Deploy fix, publish post-incident report

8. Patch Management

Automated CI/CD pipelines (GitHub Actions + cloud deployment) ensure security patches are tested and deployed rapidly. Operating systems and runtime environments are maintained by cloud providers with automatic patching.

9. Data Retention & Deletion

  • User data is retained only for the duration of the active service agreement.
  • Upon account deletion or contract termination, all associated data is permanently deleted within 30 days.
  • Users may request data export or deletion at any time.

10. Compliance

DragonBall ERP is committed to complying with applicable data protection regulations, including GDPR principles and TikTok Shop platform policies. This policy is reviewed and updated at least annually.
